# opendray

> opendray is a self-hosted control gateway for AI coding CLIs (Claude Code, Codex, Gemini CLI). It wraps local CLIs and exposes them via REST + WebSocket so web admin, mobile apps, messaging platforms, and your own apps can all drive the same session. One Claude Pro subscription serves your entire personal-app stack instead of per-token billing.

Status: v1.0 stable. Single Go binary. Self-hosted. MIT licensed.

For AI agents: this site exposes machine-readable artifacts at:
- https://docs.opendray.dev/llms.txt — this file
- https://docs.opendray.dev/llms-full.txt — concatenated full text of every page
- https://docs.opendray.dev/openapi.yaml — REST API spec

- https://docs.opendray.dev/mcp-manifest.json — connect via Model Context Protocol

- [环境记忆 — 捕获规则](https://docs.opendray.dev/zh/ambient-memory/capture-rules) [stable]: 规则 = 匹配(provider / cwd glob / 事件类型)+ 动作(summarize / regex 提取)。每事件可多规则。在 Settings → Ambient Memory 配置。
- [环境记忆 — 注入档案](https://docs.opendray.dev/zh/ambient-memory/injection) [stable]: 档案 = 注入到 session 初始上下文的内容。3 种模式 — dense(top-K 通过 search)、sparse(仅最近)、off。按 (provider, cwd) 匹配。
- [环境记忆 — 概览](https://docs.opendray.dev/zh/ambient-memory/overview) [stable]: 自动 capture 引擎 — 从 session 输出提取事实,不需要 agent 自己调 memory_store。基于规则 + LLM 摘要。写到 project/global scope。

- [备份 — 导出(Plan C)](https://docs.opendray.dev/zh/backup/export) [stable]: /export = 可移植 zip-bundle。解码数据(JSON/markdown),不是 pg_dump。可导入到不同 opendray 主机。AES-GCM 加密。
- [备份 Backup — 概览](https://docs.opendray.dev/zh/backup/overview) [stable]: 两个安全网 — 加密 pg_dump(全 DB,原地恢复) + zip-bundle 导出(纯数据,跨主机迁移)。两者都 AES-GCM 加密。Opt-in 需要 OPENDRAY_BACKUP_ENABLED。
- [备份 — 快速开始](https://docs.opendray.dev/zh/backup/quickstart) [stable]: 默认关闭。设 OPENDRAY_BACKUP_ENABLED=1 + OPENDRAY_BACKUP_KEY(32 字节 base64)。重启 → Backups 页出现。加 target → run 一次 dump。
- [备份 — 恢复(A) 和 导入(C)](https://docs.opendray.dev/zh/backup/restore-and-import) [stable]: /backups → 选 dump → Restore(原地 pg_restore)。/export → 上传 .zip.enc → Import(解密 + 合并)。都需要 OPENDRAY_BACKUP_KEY。
- [备份 — 调度](https://docs.opendray.dev/zh/backup/schedules) [stable]: cron 表达式触发备份 run。扇出到所有 enabled target。每次成功上传后跑 retention 修剪。默认 `0 2 * * *`(每天凌晨 2 点)。
- [备份 — Targets](https://docs.opendray.dev/zh/backup/targets) [stable]: 加密 bundle 落到哪。8 种 target — local-fs / S3 / R2 / B2 / GCS / Azure Blob / SMB / SFTP。每个独立 retention + cron。
- [Bridge — 通过 WebSocket 接入自定义平台](https://docs.opendray.dev/zh/channels/bridge) [stable]: Channels → New → kind=bridge → 保存 WS URL + token → 用任何语言跑一个 adapter。Adapter 说 opendray 的 WS 协议,opendray 把它当普通 channel。
- [钉钉 DingTalk](https://docs.opendray.dev/zh/channels/dingtalk) [stable]: 钉钉群加自定义群机器人 → 保存 webhook URL + SEC 签名密钥 → Channels → New → kind=dingtalk 粘进去。仅出站 — 没入站回复,没回调按钮。
- [Discord](https://docs.opendray.dev/zh/channels/discord) [stable]: 创建 Discord app + bot → 开 Message Content Intent → 邀请到服务器 → Channels → New → kind=discord 粘 token。Gateway WS,不需要公网 URL。
- [飞书 Feishu / Lark](https://docs.opendray.dev/zh/channels/feishu) [stable]: 创建飞书 app → 拿 App ID + Secret → 把 opendray 的 webhook URL 注册到 Event Subscriptions → 把 bot 加进群。需要公网 HTTPS URL。

- [Slack](https://docs.opendray.dev/zh/channels/slack) [stable]: 创建 Slack app → 开 Socket Mode → 拿 xoxb + xapp token → Channels → New → kind=slack 粘进去。不需要公网 URL。
- [Telegram](https://docs.opendray.dev/zh/channels/telegram) [stable]: 从 @BotFather 拿 bot token,粘到 Channels → New → kind=telegram 即可。long-poll,不需要公网 URL。

- [启动会话](https://docs.opendray.dev/zh/sessions/spawning) [stable]: Sessions → + New → 选 provider + cwd +(可选)name / args / claude_account / bypass / parent_session → Spawn。opendray 验证 cwd,fork PTY,首字节(或 500ms 超时)后标记 RUNNING。
- [Tab 与键盘导航](https://docs.opendray.dev/zh/sessions/tabs) [stable]: 点击 / 拖 / 右键 tab。vim 风格 g 前缀快捷键。Ctrl+Tab 切换、Ctrl+1-9 跳转、Ctrl+W 关闭、n s 打开 Spawn、g i 切换 Inspector。
- [设置 — 通用](https://docs.opendray.dev/zh/settings/general) [stable]: 网络绑定(host/port/TLS)、admin 密码、locale 默认。网络变更需重启;admin 密码立即生效。
- [设置 — 键盘 & 主题](https://docs.opendray.dev/zh/settings/keyboard-and-theme) [stable]: 每用户(浏览器 local)。主题 — light/dark/system。键盘 — vim/emacs/none。持久化到 localStorage,跨 tab 同步。
- [设置 — 日志](https://docs.opendray.dev/zh/settings/logging) [stable]: 级别(trace/debug/info/warn/error/off) + 格式(json/text) + 目标(stdout/file/syslog)。SIGHUP 热加载级别。UI 里有 live tail。
- [设置 Settings — 概览](https://docs.opendray.dev/zh/settings/overview) [stable]: 3 组 — User(主题、键盘、locale)、Server(日志、存储路径、重启)、Session defaults(per-provider runtime)。能热加载的热加载;其余要重启。
- [设置 — 重启](https://docs.opendray.dev/zh/settings/restart) [stable]: 某些 config 改动(网络、存储路径、备份、记忆 backend)需要重启。Settings → Restart 有个守卫按钮,drain in-flight 后触发 SIGTERM。
- [设置 — 会话默认](https://docs.opendray.dev/zh/settings/session-defaults) [stable]: per-provider runtime 默认。claude 的 bypass_permissions / max_turns / skills。下次 spawn 生效。per-session 可覆盖。
- [设置 — 存储路径](https://docs.opendray.dev/zh/settings/storage-paths) [stable]: opendray 把文件放哪 — notes vault、runtime / session 临时、claude 账号目录、backup staging。默认 $HOME/.opendray。改动需重启。
